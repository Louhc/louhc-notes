    对拍

    Background

    你正在做一道题，代码已经写完了，但是怎么交都过不了。

    瞪眼法找不到 bug，自己编数据怎么看都没问题，OJ 的测试数据也不给你下载 QAQ

    怎么办？

    Introduction

    对拍就是用来解决这种情况的。它能帮助你快速找到一组会让你的代码过不了的数据。

    对拍的原理也十分简单，本质上就是不断生成数据，然后将你写的程序的输出和正确的输出进行对比，直到找到差异。

    Bat

    先来介绍一下 bat 文件。

    批处理文件，在DOS和Windows（任意）系统中，.bat文件是可执行文件，由一系列命令构成，其中可以包含对其他程序的调用。——百度百科

    所以说，实际上 bat 文件就是一堆 cmd 命令。

    Duipai

    先准备好三个文件。（注意一下，都不用文件输入输出）

    用于生成数据的程序。

    gen.cpp
    #include <bits/stdc++.h>
using namespace std;

int main(){
    srand(time(0)); // 千万千万千万别漏了这个
    /*
        生成数据
    */
    return 0;
}

    你要检查的程序。

    code.cpp
    #include <bits/stdc++.h>
using namespace std;

int main(){
    /* ... */
    return 0;
}

    你写的暴力算法（保证正确）。

    brute.cpp
    #include <bits/stdc++.h>
using namespace std;

int main(){
    /* ... */
    return 0;
}

    然后写一个 bat 文件（可以先创建一个文本文件，然后修改扩展名为 bat）

    dp.bat
    g++ -std=c++14 gen.cpp -o gen
g++ -std=c++14 code.cpp -o code
g++ -std=c++14 brute.cpp -o brute   %这三行是将三个程序编译一遍，视情况可以去掉%
:loop                               %循环标记%
gen > data.txt                      %生成数据到 data.txt%
code < data.txt > out.txt           %输入 data.txt，输出到 out.txt%
brute < data.txt > ans.txt          %输入 data.txt，输出到 ans.txt%
fc out.txt ans.txt                  %比较 out.txt 和 ans.txt%
if errorlevel 1 pause               %如果不一样就暂停%
goto loop                           %回到 loop 继续循环%

    然后点击 dp.bat 运行对拍，当对拍停下来的时候，你需要的数据就在 data.txt 里面了。

    你可以尝试减小数据的规模，虽然这样可能需要花费更长的时间找到合适的数据，但是会更容易debug。
